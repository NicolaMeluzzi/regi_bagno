I comandi del DDL

Data Definition Language in SQL

Struttura, Tipi di Dato e Vincoli di Integrità

Che cos'è il DDL?

Il DDL è l’insieme dei comandi SQL usati per definire la struttura del database.

  • CREATE: Crea nuovi database o tabelle.
  • ALTER: Modifica strutture esistenti.
  • DROP: Elimina database o tabelle.

CREATE & DROP DATABASE

Spiegazione

CREATE crea il contenitore principale. DROP lo elimina definitivamente insieme a tutti i dati.

Esempio

CREATE DATABASE Scuola;
-- Per eliminare:
DROP DATABASE Scuola;

CREATE TABLE e Tipi di Dato

Permette di creare una tabella specificando campi, tipi e vincoli.

CREATE TABLE Studenti (
    ID_Studente INT PRIMARY KEY,
    Nome VARCHAR(50) NOT NULL,
    Cognome VARCHAR(50) NOT NULL,
    Eta INT CHECK (Eta >= 6 AND Eta <= 19)
);
Esercizio: Prova a creare una tabella con almeno 3 campi e un vincolo personalizzato.

Modificare ed Eliminare Tabelle

ALTER TABLE

Aggiunge, elimina o cambia colonne in una tabella già esistente.

ALTER TABLE Studenti
ADD Email VARCHAR(100);

DROP TABLE

Elimina completamente la tabella dal database.

DROP TABLE Studenti;

Slide 10: Presenza e Unicità

NOT NULL: Obbliga a inserire un valore. Niente campi vuoti.

UNIQUE: Assicura che ogni valore sia unico (es. Email).

Slide 11: Identità e Validazione

PRIMARY KEY: Identifica ogni riga in modo univoco. È la combinazione di NOT NULL e UNIQUE.

CHECK: Impone una condizione logica sui dati inseriti.

Esempio: CHECK (Eta >= 18)
