SQL Constraints

Integrità e Sicurezza del Dato

Presentazione Tecnica - Slide 9-12 + Sitografia

Slide 9: Perché i Vincoli?

I vincoli assicurano l'Integrità dei Dati.

  • Evitano errori umani durante l'input.
  • Mantengono la coerenza tra diverse tabelle.
  • Rendono il database affidabile nel tempo.

Slide 10: Presenza e Unicità

NOT NULL: Forza la presenza di un dato. Impedisce campi "fantasma".

UNIQUE: Ogni riga deve essere diversa. Ottimo per Email o Username.

Slide 11: Identità e Validazione

PRIMARY KEY: È il DNA della riga. Unisce NOT NULL e UNIQUE.

CHECK: Definisce una regola logica.
Esempio: Prezzo > 0 o Età >= 18.

Slide 12: Applicazione Pratica


CREATE TABLE Utenti (
    ID int PRIMARY KEY,       -- Identificativo Unico
    Nome varchar(50) NOT NULL,-- Obbligatorio
    Email varchar(50) UNIQUE, -- Non duplicabile
    Eta int CHECK (Eta >= 18) -- Regola personalizzata
);
